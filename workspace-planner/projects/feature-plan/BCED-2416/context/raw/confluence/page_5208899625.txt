Background and Prerequisite Knowledge

This feature addresses the absence of a "Cancel" button in various dashboard and
report execution workflows. When users encounter performance issues during
dashboard or report execution, having the ability to cancel the process is a
critical need. This feature introduces a Cancel button in the GUI, enabling
users to instantly terminate ongoing dashboard/report executions along with
their underlying SQL jobs, ensuring better control and an improved user
experience.


Capabilities in 25.08

In summary, Library Web offers complete functionality for cancelling executions.
In contrast, Workstation has certain limitations: (1) There is no dedicated
cancel button for dashboard execution, but clicking the close button may cancel
jobs; (2) There is also no cancel button to stop the "apply prompts" process in
dashboards on Workstation.

Feature flag

The feature is on by default since 25.08, here are the steps to disable the
feature:

 1. Access the machine where MicroStrategy Library is installed or deployed, and
    navigate to the following path: <Library Folder
    Path>/WEB-INF/classes/config.

 2. Locate and edit the configOverride.properties file. Add the following line:
     
    features.cancel-execution.enabled=false

features.cancel-reverse-undo-dashboard-authoring.enabled = false

 3. Save and close the file.

 4. Restart the application server.

1. Purpose

We have several update on Library PDF export editor, adding new features like
smart default options, updating/revising new strings and providing more
tooltips, improve accessibility.  Not all of these modifications are applied to
other client's PDF export editor. This feature aims to make WorkStation client'
PDF editor looks like the same and behave the same.


2. Design

We will applied the updates to workstation live/local mode's PDF export editor.
But since it reuses BIWeb's code and style, most of the changes are on BIWeb
side. So the inside Dossier's entries on BIWeb will also be changed to be the
same with WorkStation.

2.1.3 WORKSTATION LIVE MODE

Workstation packages BIWeb's code. The modifications made to BIWeb will be
automatically be applied to workstation. This means that we do not need to care
about modifications which only affect the PDF Export Editor, like smart
defaults, tooltips and strings update. We need some customization when sending
options from editor to iServer because workstation live mode deals with Library
server while BIWeb deals with web server.

We already handles two new options "Expand all grid data" and "Show table of
contents" for workstation live mode.  There should be no work left for this
client.

2.1.4 WORKSTATION OFFLINE MODE

As described in above section, we only need to take care of new options. But
instead of sending new options to iServer, offline mode handles them on its own.

2.1.4.1 EXPAND ALL GRID DATA

The main logic of this feature is written in JavaScript which workstation local
mode would use directly. The work here is to pass the option from PDF Export
Editor to JS code.

 * expose "Expand all grid data" on PDF Export Editor. BIWeb hides this option
   on purpose for offline mode, we need to remove the special logic to always
   show this option
 * Parse this option from editor and pass it to JS code. This is done in
   workstation-engine component. We need to add a new field in structure
   PDFExportOptions to save and pass the option. This should be done in both Mac
   and Windows platform.

2.1.4.2 SHOW TABLE OF  CONTENTS

We do not support this option in offline mode. New Export Engine adds TOC to PDF
via its Merger component and this is implemented with AdobeSDK using Java
Language. Workstation implements and maintains its own merger. On mac platform,
this merger is written in Object-C. In windows platform, this merger is written
in C# with AdobeSDK (C# edition). To support this option, we need to port the
logic from Java to C# on Windows platform. But on Mac Platform, we need to
rewrite the logic in Object-C.  The effort and risk are both high.  We decide
not to support this option in offline mode for now.

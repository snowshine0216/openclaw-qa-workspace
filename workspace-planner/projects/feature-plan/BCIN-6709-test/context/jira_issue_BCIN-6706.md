  # Ability to view SQL of failed grid report with Workstation. Or continue editing similar to how Developer allows   
                                                                                                                      
  ⏱️  Mon, 22 Dec 25  🔎 Jira Integration  🚀   📦 None  🏷️  None  👀 0 watchers                                      
                                                                                                                      
  ------------------------ Description ------------------------                                                       
                                                                                                                      
  **Description:** As part of validating workstation for broader rollout to users, I have run into some issues or     
  enhancement requests.  Please see attached list. I want to check if we can get on call to review this consolidated  
  list and open additional case(s) as needed.                                                                         
                                                                                                                      
  **Expected Behavior:** Fill the gap for Developer/Workstation parity.                                               
                                                                                                                      
  **Steps to Reproduce:** 1. Launch Workstation and login to an environment.                                          
                                                                                                                      
  2. Try creating a new report from scratch and execute.                                                              
  3. Notice the report execution failed, but there is no easy way to find out at which step, the error occurred       
  similar to how Developer's Report Editor ❭ SQL view suggests                                                        
                                                                                                                      
  **Business Impact:** Currently, Workstation does not allow users to view the SQL of failed grid reports or continue 
  editing them, as is possible with Developer. Without this capability, users are unable to efficiently diagnose and  
  resolve report failures, leading to increased troubleshooting time and delayed access to critical business data.    
  This limitation reduces productivity and may require reliance on additional tools or support resources to address   
  issues that could otherwise be resolved directly within Workstation.                                                
                                                                                                                      
  **Troubleshooting Steps:** Dec 19th - Customer noticed that Ability to view SQL of failed grid report with          
  Workstation is not available unlike how Developer offers

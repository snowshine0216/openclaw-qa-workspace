 Great catch — I reviewed the BCIN-976 and BCIN-7289 run folders plus the report generator code, and your diagnosis is correct.                                                                                                                                               
                                                                                                                                                                                                                                                                              
 What went wrong                                                                                                                                                                                                                                                              
                                                                                                                                                                                                                                                                              
 ### 1) BCIN-150 shows “0 open defects” but you see 1 linked defect                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                              
 Root cause is in phase2.sh defect extraction logic.                                                                                                                                                                                                                          
 It currently queries defects only by:                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                              
 - parent = "<feature-key>" OR                                                                                                                                                                                                                                                
 - text ~ "<feature-key>"                                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                              
 That misses defects linked through issue links / indirect links / other relation fields (common in your Jira flow).                                                                                                                                                          
 So the pipeline can undercount and report 0 even when Jira UI shows linked defects.                                                                                                                                                                                          
                                                                                                                                                                                                                                                                              
 ────────────────────────────────────────────────────────────────────────────────                                                                                                                                                                                             
                                                                                                                                                                                                                                                                              
 ### 2) BCIN-976 area classification is wrong (e.g., BCIN-7775 → “Image Handling”)                                                                                                                                                                                            
                                                                                                                                                                                                                                                                              
 Root cause is in derive_functional_area.mjs + data passed from build_feature_summary.mjs:                                                                                                                                                                                    
                                                                                                                                                                                                                                                                              
 - classifier scans raw summary+description text                                                                                                                                                                                                                              
 - BCIN-7775 description contains an embedded attachment image-...png in ADF payload                                                                                                                                                                                          
 - keyword rule /image/ falsely classifies it as Image Handling                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                              
 So this is a false positive from attachment metadata, not actual defect intent.                                                                                                                                                                                              
                                                                                                                                                                                                                                                                              
 ────────────────────────────────────────────────────────────────────────────────                                                                                                                                                                                             
                                                                                                                                                                                                                                                                              
 ### 3) Sections 7/8/9/10 look templated / repeated                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                              
 Confirmed partially true:                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                              
 - Section 9 and Section 10 are currently near-static template text with minimal variable injection.                                                                                                                                                                          
 - Section 7/8 are data-driven, but quality is weak because upstream area classification is noisy and one aggregation bug exists:                                                                                                                                             
     - in area table logic, “high” counts can include closed items, causing weird lines like “1 unresolved defects (3 high)”.                                                                                                                                                 
                                                                                                                                                                                                                                                                              
 So it’s not fully hardcoded, but effectively behaves generic due logic quality issues.

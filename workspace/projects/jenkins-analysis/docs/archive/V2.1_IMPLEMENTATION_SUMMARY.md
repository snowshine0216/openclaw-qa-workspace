## Issues Fixed

### ✅ Issue 1: Sort/Subtotals Parser (5 min)

**Problem:** Parser returned 0 failures for Sort #888 and Subtotals #919

**Root Cause:** TC headers had `[0-0]` prefix + emoji before `[TC...]` pattern:
```
Actual:   [0-0] 👉👇 [TC85322] Report editor sort TC85322
Expected:   [TC85322] Report editor sort TC85322  
```

**Fix:** Updated `parser_v2.js` line 230 regex:
```javascript
// Before
const tcBlocks = content.split(/(?=\s*\[?TC\d+\])/m).filter(b => b.trim() !== "");

// After
const tcBlocks = content.split(/(?=.*?\[TC\d+)/m).filter(b => {
  const trimmed = b.trim();
  return trimmed !== "" && trimmed.includes('[TC');
});
```

**Test Results:**
- Sort #888: 1 failure extracted (TC85322) ✅
- Subtotals #919: 1 failure extracted (TC85744) ✅

### ✅ Issue 3: Historical Tracking Fixed (8 min)

**Problem:** `last_failed_build` showed NULL instead of previous build number

**Root Causes Found:**
1. **Wrong parameter passed:** Called `processStep(db, failedJobId, triggerJobName, ...)` but should use actual job name
2. **Wrong SQL join:** Queried `jr.job_build` (trigger build) instead of `fj.job_build` (actual job build)

**Fixes:**
1. `db_writer.js` line 240: Changed `triggerJobName` → `jobInfo.name`
2. `db_writer.js` line 115: Changed query to use `fj.job_build` and removed `JOIN job_runs`

**Before Query:**
```sql
SELECT jr.job_build FROM failed_steps fs
JOIN failed_jobs fj ON fs.failed_job_id = fj.id
JOIN job_runs jr ON fj.run_id = jr.id
WHERE fj.job_name = ? AND fs.error_fingerprint = ? AND jr.job_build < ?
```

**After Query:**
```sql
SELECT fj.job_build FROM failed_steps fs
JOIN failed_jobs fj ON fs.failed_job_id = fj.id
WHERE fj.job_name = ? AND fs.error_fingerprint = ? AND fj.job_build < ?
```

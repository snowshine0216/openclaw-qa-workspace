OpenClaw · AI-Driven QA Platform

QA Agent Architecture

A five-agent autonomous ecosystem covering the full QA lifecycle — monitoring, planning, execution, self-healing, and reporting. 🔮 An Agent Evaluation cycle is planned as a future phase.

💰
Investment & Cost
  • AI Model (Sonnet-4.5): ~$3–15/Mtoken via GitHub Copilot; estimated $200–800/mo at current scale
  • Infrastructure: pm2 daemon on existing CI server — minimal additional hosting cost; ~$0 marginal
  • Playwright CLI: Open-source — zero licensing cost; runs on existing CI nodes
  • Atlassian Jira/Confluence: Already licensed — reusing existing seats (no new costs)
  • Engineering Setup: ~3–4 weeks initial build (5 agents + workflows + skills) by 1–2 engineers
  • Ongoing Maintenance: ~2–4 hrs/week for prompt tuning, skill updates, and edge-case handling
🚀
Business Value & ROI
  • QA Plan Automation: Cuts plan drafting from ~4 hrs → ~20 min per feature; saves ≥3 hrs/feature
  • Zero-Touch CI Healing: Auto-heals locator drift in Playwright scripts — reduces flaky test toil by ~70%
  • WDIO → Playwright Migration: Agent-assisted migration accelerates script conversion by 3–5× vs. manual
  • Defect Monitoring: Automated Jira polling catches regressions faster, reducing bug-escape rate to production
  • QA Summary Reports: Fully automated Confluence updates eliminate manual reporting (~2 hrs/sprint)
  • Scalability: System scales across any number of features simultaneously without adding QA headcount

Key Operational Scenarios
1
Daily — Automation Failure (CI)
  • 1CI job fails; webhook fires → Daily Agent
  • 2Daily Agent passes context → Orchestrator → Tester
  • 3Tester auto-heals locator drift → PR raised
  • 4True bug? Tester → Reporter → Jira ticket filed
2
Daily — Feature Defects Status Monitor
  • 1pm2 cron polls Jira for defect status changes
  • 2Daily Agent reports state changes → Orchestrator
  • 3Reporter updates defect analysis doc
  • 4Confluence QA Summary refreshed automatically
3
Daily — Defects Assigned (Full Lifecycle)
  • 1Jira ticket → "Ready for QA"; cron catches event
  • 2Orchestrator invokes Planner → QA Plan drafted
  • 3Tester writes & runs Playwright scripts
  • 4Reporter updates Jira & Confluence with results
4
Manual — Invoke Planner (QA Plan Phase)
  • 1Engineer manually triggers planning workflow
  • 2Orchestrator directly invokes Planner
  • 3Planner generates high-level QA plan + test cases
  • 4Markdown output stored for Tester consumption
5
Manual — Invoke Reporter (QA Summary Phase)
  • 1Engineer manually triggers reporting workflow
  • 2Orchestrator directly invokes Reporter
  • 3Reporter aggregates results, defects & QA plan
  • 4Final QA Summary published to Confluence
6
🔮 Agent Evaluation Cycle (Not Yet Started)
  • 1Weekly cron or manual trigger fires post-sprint
  • 2Orchestrator collects structured outputs from all agents
  • 3AI evaluation scores Planner, Tester, Reporter & Daily
  • 4Findings → Confluence report + Feishu notification
  • 5Insights feed back into prompt & skill tuning
